CLEAR CACHE – Automated and Manual

  • AUTOMATED Clear Cache

    • The Clearcache script can be found on the ecom Stager(lxomavtecom00) at - /export/apps/clearcacheWLS/clearcache.sh

  1. Login to Ecom stager
  2. Sudo to desired application (weblogic only)
  3. Execute the ClearCache Script /export/apps/clearcacheWLS/clearcache.sh
  4. Type in the Alias ID of the desired application
  5. The list of nodes will be printed, select any one of them or type 'all' to clear the cache on all nodes.
  6. A confirmation message will be printed and you're done !

NOTE:

The script logs all operations performed. We will make the script display the logfile's absolute path very soon.We will try and get it integrated in the WLSDEPLOY script as well.

    MANUAL Clear Cache with Bounce Steps

    Hard Bounce :- Stopping all the servers at the same time and then restarting all the servers.

    Rolling Bounce :- Stopping and starting the servers (i.e) Admin,managed servers one by one

    Hard Bounce with cache clearance:

At times bounce is asked with cache clearance. In such cases, STOP the application first.

Stopping the application

  1. Sudo to the application from stager.
  2. Give wlsdeploy.
  3. Select “7”
  4. Ensuring all the servers are down.

Cache clearance

On Admin server:

App_alias-id (qcordering3 for qcordering in ITV3 environment)

ls -ltr

rm -rf tmp

App alias-id

cd AdminServer

ls -ltr

rm -rf cache tmp

On managed server:

App_alias-id

cd servers

ls -ltr

rm -rf app alias1/app alias2 (qcordering32 for qcordering ITV3 environment and managed server2)

START the application.

  1. From the stager give wlsdeploy
  2. Enter the option as “8”
  3. Ensure all the servers are up.

Rolling Bounce with cache clearance:

  1. Stop one server by choosing managed stop in the wlsdeploy; then chose the server name displayed
  2. After ensuring the application is down in that server perform the cache clearance

On managed server:

App_alias-id

cd servers

ls -ltr

rm -rf appalias1/appalias2 (qcordering32 for qcordering ITV3 environment and manged server2)

  1. Ensure the application is up in the managed server only then move to the next server
  2. Now repeat step-1,2,3 in all managed servers.
